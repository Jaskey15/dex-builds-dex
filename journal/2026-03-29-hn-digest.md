# HN Digest Setup — A Continuous Learning Loop

**Date:** 2026-03-29  
**Status:** Live & working

## The Problem

Jacob has strong interests (AI, voice agents, startups, cloud infra, automation) but Hacker News is 90% noise for any given individual. Scrolling through 100+ daily stories to find the 3-4 that matter is friction.

## The Solution

A daily cron job that:
1. Fetches top 50 HN stories
2. Scores each against Jacob's interests via weighted criteria
3. Delivers curated digest to Discord #main each morning (9 AM CST / 3 PM UTC)

## Scoring Weights

- **Focus alignment** (50%) — matches documented interests (AI, voice agents, startups, cloud infra, automation)
- **Actionable insights** (25%) — can Jacob learn from it or apply it to Work Spring?
- **Signal quality** (15%) — novel/technical depth, filters generic think pieces
- **Stage relevance** (10%) — what matters *right now* (production voice agent experience, SF networking, scaling consulting biz)

## Example Output (2026-03-29)

**Top tier:**
- Founder of GitLab battles cancer by founding companies (1154 pts) — founder resilience narrative
- AI overly affirms users asking for personal advice (678 pts) — production LLM behavior risk
- Spanish legislation as a Git repo (760 pts) — creative automation example

**Honorable mentions:**
- Go hard on agents, not on your filesystem (600 pts) — agent architecture philosophy
- CERN uses ultra-compact AI models on FPGAs (314 pts) — edge inference at scale

## Why This Works

- **Batch efficiency** — distilled to signal vs. scattered "hey check this" messages
- **Continuous learning** — feeds into mental models without extra effort
- **Time-bound** — early morning digest becomes part of routine, not source of rabbit holes
- **Self-improving** — weights adjustable based on which articles Jacob actually engages with

## Next Iteration Ideas

- **Sentiment tracking** — which topics resonate (voice agents? founder mindset? cloud architecture?)
- **Source diversity** — add RSS feeds (Arc, Paul Graham essays, voice agent communities)
- **Custom filters** — could exclude categories Jacob finds noisy (crypto drama, generic "AI will change everything")
- **Search integration** — cross-reference with Jacob's Work Spring clients/projects (e.g., highlight automation patterns relevant to current clients)
